# 農家×学生マッチング UX基盤整理

## ペルソナ概要

- **農家（田中さん・50代）**
  - 課題：繁忙期の人手不足、短期間で信頼できる学生を確保したい。
  - ゴール：募集の公開から承認までをスマホで完結し、作業当日をスムーズに迎える。
- **学生（佐藤さん・20代）**
  - 課題：農業体験の情報が点在し、応募〜参加後の実績管理が面倒。
  - ゴール：興味ある募集を素早く見つけ、応募〜チェックイン〜マイル管理まで直感的に操作。
- **運営（中村さん・30代）**
  - 課題：募集の品質管理と参加者フォローを限られたリソースで回す必要。
  - ゴール：応募状況を俯瞰し、承認・連絡をテンプレ化して迅速に対応。

## シナリオ（代表導線）

1. 学生がアプリにログインし、ホームでおすすめ募集を閲覧。
2. 気になる募集詳細を開き、条件を確認後「応募」→即時に「応募済」表示へ切り替え。
3. 運営ボットからチャットで承認通知が届き、参加前ガイドを確認。
4. 当日、学生は現地でQRチェックイン→作業完了後にマイルが付与されグラフで可視化。
5. マイルを活用し、スタジオ予約カレンダーから希望枠をタップ→トーストで予約完了を演出。

## 画面遷移サマリ

- ローンチ → ペルソナ選択付きオンボーディング → Firebase Auth匿名ログイン。
- ホームタブ：おすすめ募集 / マイル残高サマリ。
- 募集一覧 → フィルタモーダル（作業カテゴリ、エリア、日程フィルタ）。
- 募集詳細 → 応募ボタン → 応募済ステータス + チャット導線。
- チャット（学生・運営ボット） → 作業ガイドリンク → チェックイン画面。
- チェックイン画面：QRスキャン or GPSボタン → 作業進捗表示。
- マイルダッシュボード：獲得履歴一覧 + `fl_chart`で推移表示。
- 予約カレンダー：`TableCalendar`上の空きピン → 予約完了トースト。
- 設定：プロフィール、チュートリアル再閲覧、ログアウト。

## Flutterテーマ方針（Material 3）

- **カラーパレット**
  - Primary: `#3B7A57`（農地の緑）
  - Secondary: `#F4A261`（土壌・収穫を想起）
  - Tertiary: `#2A9D8F`（水・共創の爽やかさ）
  - Surface: `#F8F9F4`
  - Error: Material 3デフォルト `#B3261E`

- **タイポグラフィ**
  - Display Large：Noto Sans JP Semibold 36
  - Title Medium：Noto Sans JP Medium 20
  - Body Medium：Noto Sans JP Regular 16
  - Label Large：Noto Sans JP Medium 14（ボタン）

- **コンポーネントスタイル**
  - カード角丸：16dp、Elevation 2
  - チップ：8dp角丸、フィルタ状況を色で提示（オン=Primary Container）
  - ボタン：Filled Buttonを基本、Secondary色でTonalボタンを用意
  - Snackbar/Toast：Rounded 12dp、背景`#2A2A2A`、アクションテキストはPrimary

- **アニメーション**
  - スクリーン遷移：`PageRouteBuilder`でFade + Slide (200ms)
  - FABや主要ボタン：Scaleトランジションでタップレスポンスを演出

## アセット・トーン

- 写真：「農作業中」「収穫物」「笑顔の学生」などCC0素材を想定。`assets/images/`管理。
- アイコン：Material Symbols Roundedを優先使用。
- イラスト：オンボーディング用に柔らかいフラットイラスト（Figmaライブラリ利用）。
